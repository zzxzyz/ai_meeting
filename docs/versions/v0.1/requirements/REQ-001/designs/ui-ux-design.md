## 1. 设计概述

### 1.1 设计目标

- **简洁高效**: 最少步骤完成注册/登录
- **清晰友好**: 提示信息准确、易懂
- **品牌一致**: 符合视频会议平台的专业形象
- **响应式**: 支持不同屏幕尺寸
- **无障碍**: 符合 WCAG 2.1 AA 标准

### 1.2 设计原则

1. **表单优先**: 减少干扰,聚焦核心操作
2. **即时反馈**: 实时验证,及时提示错误
3. **进度可见**: 加载状态清晰可见
4. **容错设计**: 友好的错误提示和恢复路径
5. **品牌温度**: 专业但不失亲和力

## 4. 表单验证规则

### 4.1 邮箱验证

**验证时机**: 失去焦点 (onBlur) + 提交时

**验证规则**:
- 非空
- 符合邮箱格式(正则: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`)

**错误提示**:
- 为空: "请输入邮箱地址"
- 格式错误: "请输入有效的邮箱地址"

### 4.2 昵称验证

**验证时机**: 实时 (onChange) + 提交时

**验证规则**:
- 非空
- 长度 2-20 个字符
- 不允许纯空格
- 支持中文、英文、数字、下划线

**错误提示**:
- 为空: "请输入昵称"
- 长度不符: "昵称长度为 2-20 个字符"
- 包含非法字符: "昵称仅支持中文、英文、数字、下划线"

### 4.3 密码验证

**验证时机**: 实时 (onChange) + 提交时

**验证规则**:
- 非空
- 最小长度 8 个字符
- 包含字母 (a-z 或 A-Z)
- 包含数字 (0-9)

**错误提示**:
- 为空: "请输入密码"
- 长度不足: "密码至少 8 个字符"
- 缺少字母: "密码必须包含字母"
- 缺少数字: "密码必须包含数字"

### 4.4 表单提交验证

**验证顺序**:
1. 前端验证(立即反馈)
2. 后端验证(最终校验)

**提交按钮状态**:
- 默认禁用(灰色)
- 所有字段验证通过后激活(蓝色)
- 点击后显示加载状态

### 10.3 代码示例

#### Input 组件

```tsx
interface InputProps {
  label: string;
  type?: 'text' | 'email' | 'password';
  value: string;
  onChange: (value: string) => void;
  error?: string;
  icon?: React.ReactNode;
  placeholder?: string;
  required?: boolean;
}

export const Input: React.FC<InputProps> = ({
  label,
  type = 'text',
  value,
  onChange,
  error,
  icon,
  placeholder,
  required
}) => {
  return (
    <div className="space-y-2">
      <label className="text-sm text-gray-600">
        {label}
        {required && <span className="text-red-500">*</span>}
      </label>
      <div className={cn(
        "relative",
        error && "animate-shake"
      )}>
        {icon && (
          <div className="absolute left-3 top-1/2 -translate-y-1/2 text-gray-400">
            {icon}
          </div>
        )}
        <input
          type={type}
          value={value}
          onChange={(e) => onChange(e.target.value)}
          placeholder={placeholder}
          className={cn(
            "w-full h-12 px-4 rounded-lg border",
            "focus:outline-none focus:ring-2 focus:ring-primary/20",
            icon && "pl-10",
            error ? "border-red-500" : "border-gray-300"
          )}
          aria-invalid={!!error}
          aria-describedby={error ? `${label}-error` : undefined}
        />
      </div>
      {error && (
        <p
          id={`${label}-error`}
          className="text-sm text-red-500"
          role="alert"
        >
          ⚠️ {error}
        </p>
      )}
    </div>
  );
};
```

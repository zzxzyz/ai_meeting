## 1. 设计概述

### 1.1 设计目标

- **快速发起**: 一步创建会议，最短路径获取会议号
- **轻松加入**: 输入会议号即可加入，无额外步骤
- **清晰管理**: 会议列表直观展示状态与历史
- **权限分明**: 创建者与参与者操作界面有所区分
- **风格一致**: 与 REQ-001 认证页面保持统一视觉语言

### 1.2 设计原则

1. **操作最短路径**: 创建/加入会议不超过 2 步
2. **状态即时反馈**: 会议状态、错误提示实时显示
3. **容错友好**: 无效输入给出明确指引
4. **角色感知**: 创建者与普通参与者差异化展示
5. **跨端一致**: Web 与 Electron 桌面端体验一致

### 3.3 加入会议页面

加入会议支持两种入口：首页卡片内联输入 + 独立加入页面。

#### 独立加入页面

```
┌────────────────────────────────────────────────────────────┐
│  [Logo] 视频会议平台              [用户头像] 张三  [登出]   │
├────────────────────────────────────────────────────────────┤
│                                                            │
│                      加入会议                              │
│                                                            │
│              输入 9 位会议号加入会议                        │
│                                                            │
│         ┌─────────────────────────────────────┐           │
│         │         1 2 3 - 4 5 6 - 7 8 9       │           │
│         └─────────────────────────────────────┘           │
│                                                            │
│         ┌─────────────────────────────────────┐           │
│         │              加入会议                │           │
│         └─────────────────────────────────────┘           │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**错误状态展示**：

```
│         ┌─────────────────────────────────────┐           │
│         │  8 8 8 - 8 8 8 - 8 8 8              │           │
│         └─────────────────────────────────────┘           │
│         ❌ 会议不存在，请检查会议号是否正确               │
```

#### 输入格式说明

- 自动格式化：用户输入 9 个数字，自动插入 `-` 分隔符（XXX-XXX-XXX）
- 只允许输入数字，其他字符自动过滤
- 输入满 9 位后自动触发验证（非强制提交）

#### 错误提示文案

| 错误场景 | 提示文案 |
|---------|---------|
| 格式不足 9 位 | 请输入完整的 9 位会议号 |
| 会议号不存在 | 会议不存在，请检查会议号是否正确 |
| 会议已结束 | 该会议已结束，无法加入 |
| 网络错误 | 网络连接失败，请稍后重试 |

## 4. 组件规范

### 4.1 颜色规范

| 元素 | 颜色 | 说明 |

| 危险操作 | `#DC2626`（红色）| 结束会议按钮 |

| 背景 | `#F9FAFB` | 页面背景 |

### 4.2 字体规范

| 元素 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 24px | 600 |

| 正文 | 14px | 400 |
| 辅助文字 | 12px | 400 |

### 4.3 会议号展示格式

- 格式：`XXX-XXX-XXX`
- 字体：等宽字体（`font-mono`）
- 字号：28px（详情页）/ 16px（列表）

### 4.4 状态徽章

```
🟢 进行中   →  绿色背景 + 绿色文字  圆角 pill 样式
```

### 4.5 按钮规范

| 类型 | 样式 |
|------|------|
| 主要按钮 | 蓝色填充，白色文字，圆角 8px |

| 危险按钮 | 红色填充，白色文字 |
| 禁用状态 | 灰色填充，灰色文字，不可点击 |

## 5. 响应式设计

### 5.1 Web 端断点

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| 桌面大屏 | ≥ 1440px | 创建/加入卡片并排，列表宽度 1200px |
| 桌面标准 | 1280px～1439px | 创建/加入卡片并排，列表全宽 |
| 平板 | 768px～1279px | 创建/加入卡片竖排，列表隐藏部分列 |

### 5.2 Electron 桌面端

- 最小窗口尺寸：800px × 600px
- 布局与 Web 桌面标准版保持一致
- 顶部导航栏适配 Electron 原生窗口标题栏
- 会议号输入框支持粘贴整段文字后自动提取数字

### 5.3 移动端（延后至 v0.2）

- iOS / Android 原生 App 在 v0.2 实现
- 本期不需要适配 375px 移动端布局

## 6. 错误与空状态设计

### 6.1 全局错误状态

| 场景 | 展示方式 | 文案 |
|------|---------|------|
| 网络断开 | 顶部 Toast 警告条 | 网络连接失败，请检查网络后重试 |
| 服务端错误 | 页面内错误提示 | 服务暂时不可用，请稍后重试 |
| 未登录访问 | 重定向至登录页 | - |
| Token 过期 | 自动刷新 / 重登录提示 | 登录已过期，请重新登录 |

### 6.2 加载状态

- 按钮点击后显示 Loading 旋转图标，防止重复提交
- 列表加载中显示骨架屏（Skeleton）
- 会议详情加载中显示占位块

### 6.3 空状态

| 页面 | 空状态文案 | 引导操作 |
|------|----------|---------|
| 我创建的会议（空） | 暂无会议记录，点击"创建会议"发起第一场会议 | 创建会议按钮 |
| 我参加的会议（空） | 暂未参加过任何会议，输入会议号加入 | 加入会议入口 |

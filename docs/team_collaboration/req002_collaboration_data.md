## 二、任务级数据记录

### 2.1 数据记录表

**使用说明**: Team-lead 在任务创建、分配、完成时实时更新此表

| 任务ID | 任务名称 | 创建时间 | 指定角色 | 主动分配? | 实际领取者 | 领取时间 | 完成时间 | 用时(分) | 角色匹配 | 重复领取次数 | Team-lead操作耗时(分) | 备注 |
|-------|---------|---------|---------|----------|-----------|---------|---------|---------|---------|------------|-------------------|-----|
| | | | | | | | | | | | | |

**示例行** (完成第一个任务后删除):
| #1 | [product-manager] REQ-002 需求分析 | 02-15 10:00 | product-manager | ✅ | product-manager | 02-15 10:02 | 02-15 10:25 | 23 | ✅ | 0 | 3 | |

**字段说明**:
- **任务ID**: Task ID (如 #1, #2, #3...)
- **任务名称**: 任务的 subject
- **创建时间**: TaskCreate 调用时间 (格式: MM-DD HH:mm)
- **指定角色**: subject 中 `[角色名]` 指定的负责角色
- **主动分配?**: ✅ team-lead 主动分配 / ❌ agent 自主领取
- **实际领取者**: 最终完成任务的 agent 角色名 (不含 -2 等后缀)
- **领取时间**: 任务状态变为 in_progress 的时间
- **完成时间**: 任务状态变为 completed 的时间
- **用时(分)**: 完成时间 - 领取时间 (分钟)
- **角色匹配**: ✅ 匹配 (指定角色 = 实际领取者) / ❌ 不匹配
- **重复领取次数**: 任务 owner 被修改的次数 (首次分配不算)
- **Team-lead操作耗时**: 从创建任务到发送通知的总耗时 (分钟)
- **备注**: 其他需要记录的信息 (如跨角色原因、问题等)
